## 2つの拡張機能、2つの信頼モデル

Kite-Uには異なる目的を持つ2つのブラウザ拡張機能があります：

| | Context Bro | Kite-U Extension |
|---|---|---|
| **制御モデル** | ユーザー制御 | AI制御 |
| **例え** | ウェブクリッパー（何を共有するか自分で決める） | ブラウザ自動化（AIが操作する） |
| **信頼レベル** | ユーザー信頼 — すべての操作をあなたがトリガー | Agent信頼 — AIが自律的に行動可能 |
| **データフロー** | あなたのブラウザ → あなたのAPIエンドポイント | Kite-Uプラットフォーム ↔ あなたのブラウザ |
| **権限** | 読み取り専用（ページ内容の抽出） | 読み書き（クリック、ナビゲーション、フォーム入力） |

## Context Broを使うとき

- **AIアシスタントにウェブコンテキストをオンデマンドで提供**したいとき
- **何をいつ共有するか完全に制御**したいとき
- ウェブデータを入力として必要とするカスタムAIパイプラインを構築しているとき
- AI処理用に**ライブ配信チャット**をキャプチャしたいとき

## Kite-U Extensionを使うとき

- AIに**代わりにウェブを閲覧**してほしいとき
- AIに**フォーム入力、ボタンクリック、ナビゲーション**をしてほしいとき
- ウェブアプリと対話する自律Agentワークフローが必要なとき

## 両方使える？

はい！互いに補完し合います：

- **Context Bro** はAIに**目**を与えます（あなたが見ているものを読む）
- **Kite-U Extension** はAIに**手**を与えます（ウェブページと対話する）

それぞれ独立した拡張機能で、独立した権限、独立したリポジトリ、独立したインストールです。
