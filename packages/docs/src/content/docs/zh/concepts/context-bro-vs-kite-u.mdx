## 两个扩展，两种信任模型

Kite-U 有两个浏览器扩展，服务于不同目的：

| | Context Bro | Kite-U Extension |
|---|---|---|
| **控制模型** | 用户控制 | AI 控制 |
| **类比** | 网页剪藏（你决定分享什么） | 浏览器自动化（AI 驾驶） |
| **信任级别** | 用户信任 —— 你触发每个操作 | Agent 信任 —— AI 可自主行动 |
| **数据流向** | 你的浏览器 → 你的 API 端点 | Kite-U 平台 ↔ 你的浏览器 |
| **权限** | 只读（提取页面内容） | 读写（点击、导航、填写表单） |

## 何时使用 Context Bro

- 你想按需**向 AI 助手提供网页上下文**
- 你想完全控制**分享什么以及何时分享**
- 你正在构建需要网页数据作为输入的自定义 AI 流水线
- 你想捕获**直播聊天**供 AI 处理

## 何时使用 Kite-U Extension

- 你想让 AI **替你浏览网页**
- 你需要 AI **填写表单、点击按钮或导航**
- 你想要与网页应用交互的自主 Agent 工作流

## 可以同时使用吗？

可以！它们互为补充：

- **Context Bro** 给你的 AI **眼睛**（读取你正在看的内容）
- **Kite-U Extension** 给你的 AI **手**（与网页互动）

它们是独立的扩展，拥有独立的权限、独立的仓库和独立的安装。
