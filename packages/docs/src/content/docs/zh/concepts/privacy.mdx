## 允许列表优先模型

Context Bro 遵循**默认不分享**的隐私模型：

- **手动分享**可在任何页面使用（你显式触发）
- **定时自动分享**仅对你添加到允许列表的域名生效
- **不会发送任何数据**，除非你配置了端点并触发了分享操作

## 提取的数据

当你分享页面时，Context Bro 提取：

- 页面标题、URL、meta 描述、作者
- 主要内容（通过 [Defuddle](https://github.com/nicksawhney/defuddle) —— 与 Obsidian Web Clipper 相同）
- 选中文本（如有）
- Schema.org 结构化数据
- CSS 选择器查询（如果你的模板使用 `{{selector:...}}`）

## 不会收集的数据

- 无分析或遥测
- 无浏览历史
- 无密码或表单字段内容
- 无来自隐私模式标签页的数据
- 无服务端存储 —— **我们没有服务器**

## 数据发送目的地

数据只会发送到**你自己配置的 API 端点**。Context Bro 不会向任何其他目的地发起网络请求。

所有设置（端点、模板、允许列表）都通过 `chrome.storage.local` 存储在本地浏览器中。卸载扩展会删除所有数据。

## 内容去重

使用定时分享时，Context Bro 会计算提取内容的 SHA-256 哈希。如果内容自上次分享以来没有变化，则跳过该页面 —— 避免重复发送。

哈希值存储在本地，可随时在设置中清除。

## 权限说明

| 权限 | 用途 |
|------|------|
| `activeTab` | 点击分享时读取当前标签页的内容 |
| `scripting` | 向页面注入内容提取脚本 |
| `storage` | 在本地保存设置 |
| `alarms` | 驱动定时分享计时器 |
| `tabs` | 查询打开的标签页用于定时提取 |
| `contextMenus` | 添加"Share to Context Bro"右键菜单项 |
