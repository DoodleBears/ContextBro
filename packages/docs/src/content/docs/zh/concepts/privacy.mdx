## 概述

Context Bro 是一款帮助你将网页上下文分享给 AI 智能体的浏览器扩展。隐私是其设计的核心 —— 扩展遵循**允许列表优先、默认不分享**的模型。

## 我们收集的数据

**无。** Context Bro 不会在我们的服务器上收集、传输或存储任何数据。我们没有服务器。所有配置（端点、模板、站点规则）均通过 `chrome.storage.local` 存储在你的本地浏览器中。

## 你选择分享的数据

当你显式分享页面（通过弹出窗口、右键菜单、快捷键或浮动按钮）时，扩展会：

- 从当前标签页提取页面内容（标题、URL、meta 描述、作者、通过 [Defuddle](https://github.com/kepano/defuddle) 提取的正文、选中文本、Schema.org 结构化数据、CSS 选择器查询）
- 使用你配置的模板编译内容
- 将编译后的数据发送到**你自己的 API 端点** —— 一个你自己配置的 URL

除非你配置了端点并触发了分享操作，扩展不会向任何地方发送数据。

## 自动捕获（定时分享）

如果你启用了自动捕获，扩展会定期提取内容，但**仅限于你显式添加到站点规则的域名**。内容通过本地存储的 SHA-256 哈希进行去重，避免发送未变化的页面。哈希值可随时在设置中清除。

## 直播适配器

在 Twitch 和 YouTube 直播页面上，可选的适配器会观察浏览器中可见的聊天消息。聊天批次会发送到你配置的端点。适配器仅在匹配的页面上激活，可以通过不在这些站点上加载扩展来禁用。

## 不会收集的数据

- 无分析或遥测
- 无浏览历史
- 无密码或表单字段内容
- 无来自隐私模式标签页的数据
- 无服务端存储

## 第三方服务

Context Bro 不集成任何第三方分析、追踪或广告服务。唯一的网络请求仅发往你配置的端点。

## 数据保留

所有数据都存储在你的本地浏览器中。卸载扩展会删除所有存储的数据。你也可以随时通过设置清除数据。

## 权限说明

| 权限 | 用途 |
|------|------|
| `activeTab` | 点击分享时读取当前标签页的内容 |
| `scripting` | 向页面注入内容提取脚本 |
| `storage` | 在本地保存设置 |
| `alarms` | 驱动定时分享计时器 |
| `tabs` | 查询打开的标签页用于定时提取 |
| `contextMenus` | 添加"Share to Context Bro"右键菜单项 |
