## 允许列表

允许列表控制哪些域名有资格进行**定时自动分享**。只有允许列表中的域名才会被定时提取。

手动分享（弹出窗口、快捷键、右键菜单）可在**任何域名**上使用，不受允许列表限制。

### 添加域名

1. 进入 **设置 → Allowlist**
2. 输入域名模式并按 Enter
3. 单独切换每个域名的开关

### 域名模式

| 模式 | 匹配 |
|------|------|
| `github.com` | 精确域名 |
| `*.github.com` | 所有子域名 |
| `news.ycombinator.com` | 精确子域名 |

### 预设

点击预设按钮可批量添加：

| 预设 | 域名 |
|------|------|
| **Dev** | github.com, stackoverflow.com, developer.mozilla.org 等 |
| **News** | reuters.com, bbc.com, cnn.com, nytimes.com 等 |
| **Social** | reddit.com, twitter.com, mastodon.social 等 |
| **Streaming** | twitch.tv, youtube.com |

### 域名绑定模板

每个允许列表域名可以可选地绑定特定模板。当定时器提取该域名时，使用绑定的模板而非默认模板。

## 定时分享

启用后，Context Bro 会定期从允许列表域名提取页面并发送到默认端点。

### 配置

进入 **设置 → Schedule**：

| 设置 | 选项 |
|------|------|
| **启用** | 开关自动分享 |
| **间隔** | 5分钟、10分钟、15分钟、30分钟、1小时、2小时 |
| **模式** | 仅当前标签页 / 所有允许的标签页 |

### 工作原理

1. Chrome Alarms API 按配置的间隔触发
2. Context Bro 查询打开的标签页
3. 选择匹配允许列表域名的标签页
4. 对每个选中的标签页：
   - 提取内容
   - 计算 SHA-256 哈希
   - 如果内容自上次分享未变化 → 跳过（去重）
   - 否则 → 编译模板 → POST 到默认端点

### 去重

内容哈希存储在本地 `chrome.storage.local` 中。这防止了重复发送未变化的页面。清除扩展数据或卸载时会删除哈希。
